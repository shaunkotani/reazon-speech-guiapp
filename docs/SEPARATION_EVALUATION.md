# 話者別音声分離の日本語評価

## 2026-07-20 暫定ベースライン

現行の `Conv-TasNet Libri2Mix 16kHz` と `reazonspeech-k2-v2 int8` を、手元の日本語2話者会話13ケースで測定した。対象は合計81秒・参照389文字。音声と文字起こしの利用条件は外部公開用に確認できておらず、発話単位の秒精度アノテーションであるため、データは `provisional`・ローカル評価専用とした。

| 指標 | 結果 |
|---|---:|
| 混合音声 cpCER | 55.5% |
| 分離後 PIT-CER | 72.8% |
| CER改善（正なら改善） | -17.2ポイント |
| 分離前の文字カバレッジ | 50.4% |
| 品質ゲート後の文字カバレッジ | 57.6% |
| 回収できた一致文字 | 28文字 |
| 新たに増えた不一致文字 | 59文字 |
| 追加文字の一致率 | 32.2% |
| 品質ゲート採用率 | 8/13 (61.5%) |
| 分離処理 RTF | 約0.17〜0.18 |

速度は12th Gen Intel Core i7-12650H（16論理CPU）で実時間より十分速い。一方、分離後PIT-CERは混合音声より悪く、品質ゲートが追加した文字も一致28文字に対して不一致59文字だった。英語 Libri2Mix 学習の現行モデルを、日本語会話の既定機能へ昇格させる根拠にはならない。現状どおり実験機能・既定OFF・既存文字起こしを削除しないフォールバックを維持する。

この数値は暫定アノテーションに依存するため、絶対精度やリリース判定には使わない。ただし、同じマニフェストで次のモデル候補を相対比較する初期ベースラインとして利用できる。

## 2026-07-20 条件制御した合成重なり評価

同じ2話者会話の非重なり発話から48ケースを合成した。SNRは -6 / 0 / +6dB、短い方の発話に対する重なり率は25 / 50 / 75 / 100%で、各条件4ケース。合計249.4秒で、音声・正解stemはローカルにのみ保存した。

合成元stemを同じVAD・ASRへ通した結果を疑似正解にし、実際の重なりコアに接する発話だけで比較した。この指標は人手文字起こしの誤差を含まず、「単独音声なら認識できた内容を分離が保てるか」を測る。

| 指標 | 結果 |
|---|---:|
| 合成元ASRの参照文字 | 1,087文字 |
| 混合音声 cpCER | 54.6% |
| 分離後 PIT-CER | 64.1% |
| CER改善（正なら改善） | -9.6ポイント |
| 品質ゲートによる追加文字の一致率 | 24.4% |
| 分離処理 RTF | 0.150 |

12条件のうち改善したのは「SNR 0dB・重なり率100%」の4ケース（+9.5ポイント）だけだった。2条件は差なし、残る9条件は悪化した。条件当たり4ケースなので個別条件の順位はまだ確定値ではないが、全体として現行モデルが日本語発話を安定して分離できない判断は、暫定アノテーションだけの評価より強くなった。

合成評価は実会話の音響条件を再現するものではない。JVSは100話者・30時間の日本語音声を提供する候補だが、公式条件では音声利用が研究・非商用・個人利用に限定され、再配布不可で、全体ダウンロードも約3.5GBある。そのため自動取得・リポジトリ同梱はせず、利用条件を確認したローカルデータとして取り込む方針とする。

## 次の評価データ

最終判断用には、利用許諾と話者同意を確認した日本語音声を最低200重なりケース用意する。日本語会議・電話・オンライン会議・残響・声量差・短い相槌を分け、100ms以下の時刻、重なり区間内の各話者本文、二者照合を付ける。調整用と最終テストは収録・話者単位で分離する。

再現コマンドとマニフェスト仕様は [evaluation/README.md](../evaluation/README.md) に記載した。ローカルの詳細結果は `evaluation/results/current-convtasnet.json` と同名の Markdown に生成される。
